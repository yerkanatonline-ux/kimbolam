# «Кім болам?» — жоба контексі (Claude Code үшін)

> Бұл файлды Claude Code жоба қалтасын ашқанда автоматты түрде оқиды. Мұнда жобаның толық тарихы, әдіснамалық шешімдер және кодтың құрылымы жазылған — жаңа сессия осыны оқыса, бәрін қайта түсіндірусіз жалғастыра алады.

## Жоба және адам

**«Кім болам?»** — Қазақстандық мектеп оқушыларына (8–11 сынып), ата-аналарға және мұғалімдерге арналған кәсіби бағдар беру платформасы. Тапсырыс беруші — **Ерқанат Серікұлы Жақсыбаев**, профориентолог, «Кім болам?» атты кітаптың авторы (кітаптың мазмұны — 3 бөлім: Тану/Таңдау/Тұлғалық даму — платформамен біртіндеп байланыстырылып жатыр, төменде қараңыз), Instagram/Telegram/YouTube-те аудиториясы бар. Негізгі дифференциатор: «тұлға типі → сай мамандық» (бәсекелес EdUser.app-тың «ҰБТ балы → университет» логикасынан ерекшеленеді).

**Тіл:** Барлық сұхбат қазақ тілінде (кейде орысша араласады). Ерқанатпен қазақша сөйлесу керек.

**Стиль:** Ерқанат — шешім қабылдағыш, іс-қимылға бағытталған адам, БІРАҚ маңызды шешімдерде (әдіснама, дерекқор дизайны, AI-дің этикалық шектері) асығуды қатаң талап етпейді — балалардың болашағы шешілетіндіктен нәтиже дәл болуы керек деп талап етеді. Үлкен өзгерісті кодтамас бұрын жоспарды талқылап, бекітіп алу керек. Платформа қазір **өмірде, нақты адамдар тапсырып жатыр** (2026 жылдың шілде-тамызынан бастап).

## Ағымдағы жағдай (маңызды!)

Негізгі платформа (`kimbolam.html`) жұмыс істейді, дизайны толық қайта құрылды (Duolingo-стиль, төменде қараңыз). Оған қоса **төрт жаңа backend-тәуелді мүмкіндік құрылды, бірақ БӘРІ ӘЛІ ДЕПЛОЙ ЕТІЛМЕГЕН** (AI кеңесші, эссе көмекшісі, дерекқор, IDP миссиялары — төмендегі бөлімдерді қараңыз). Бұлардың ешқайсысы негізгі платформаға толық сымдалып қосылған жоқ — барлығы бөлек прототип беттерінде тексерілуде (Ерқанаттың өз талабы бойынша: «алдымен бөлек көрейік, содан кейін қосамыз»).

**Ашық қалған үлкен тапсырма:** мамандық ↔ университет байланысы. Ерқанат бұл деректі **өзі жинап, файл түрінде жібереді** (әлі жіберген жоқ). Дерек келгенде: `data/specialties-key.json`-дағы 117 кодпен байланыстырып, есеп бетіне «осы мамандықты оқытатын университеттер» бөлімін қосу керек. Дерек көзі ретінде `univision.kz` сайты тексерілді — ол дәл біздің B-код жүйесін қолданады, бірақ **B083 (Ветеринария), B120, B161** кодтары сол сайттың жалпы тізімінен табылмады.

**Кітаппен байланыс — БАСТАЛҒАН ЖОҚ әлі.** Ерқанат кітабының мазмұнын (3 бөлім: Тану/Таңдау/Тұлғалық даму) жіберуге келіскен, бірақ толық мәтінді әлі жібермеген (тек мазмұны/тарау атаулары көрілген). «5 типтік жүйе» тарауы = Климовтың 5 типі (біз қолданып жатқан жүйемен сәйкес, расталды). «ProgressMe авторлық тесті» ескірген, қолданылмайды. IDP миссия жүйесі (төменде) кітаптың III бөлімінің (Тұлғалық даму: Мақсат қою, Жоспарлау, Time management, Қаржылық сауаттылық) сандық жалғасы болу үшін жобаланды. **Ерқанат кітапты жібергенде — осы мазмұнды миссияларға толық интеграциялау керек.**

## Дизайн жүйесі (толық қайта құрылды)

Ескі "teal-navy editorial" дизайн Ерқанатқа ұнамады → мүлдем жаңа, Duolingo-стиль, "қанық түсті, 3D/ойын тектес" бағытқа көшірілді. Негізгі токендер (`kimbolam.html`-дың `<style>` блогында): `--night` (терең күлгін-түнгі, hero/footer/quote фоны), `--violet`, `--green`, `--yellow`, `--pink`, `--blue`, `--orange` (RIASEC 6 типінің әрқайсысына өз түсі), `--cream` (негізгі фон). Қаріп: **Nunito** (барлық жерде, дөңгеленген, ойыншыл, қазақ әріптерін толық қолдайды — расталды). Сигнатуралық белгі: барлық батырма/карточкада **қатты (blur жоқ) офсет көлеңке** — "3D басылатын батырма" әсері (`box-shadow: 0 Npx 0 <түс>`, `:active`-те `translateY` + көлеңке 0-ге түседі).

Барлық ескі класс атаулары (JS-тің тәуелді болатын ~50 класы, 42 DOM ID) сақталды — тек CSS қайта жазылды, HTML құрылымы негізінен сол қалпында (тек hero-ге карта-колода, меню-ге Блог/Кіру/Тіркелу, бетке дәйексөз секциясы қосылды).

## Әдіснама (ӨЗГЕРТПЕС БҰРЫН ОСЫНЫ ОҚЫ — ешқашан AI-ге ауыстырмау керек бөлік)

Толық ғылыми-әдіснамалық негіздеме: `docs/Kim_Bolam_adisnamalyq_negizdeme.docx`.

- **3 тест, 72 сұрақ:** Holland/RIASEC — 36 сұрақ · Климов ДДО — 20 жұп-таңдау (5 тип) · Юнг типологиясы (MBTI дәстүрінде, ресми MBTI ЕМЕС — авторлық құқық себебінен) — 16 сұрақ.
- **MBTI салмағы сәйкестендіруде 0%.** Мамандыққа сәйкестендіру **100% Holland+Климов**, косинус ұқсастығы + Климов «екінші пікір» түзетуі.
- **Конвергентті тексеру:** Holland топ-2 мен Климов топ-1 салыстырылады → «Жоғары»/«Орташа»/«Ізденіс керек» сенім деңгейі.
- **Норма:** `NORM_BASE` қазір бейтарап (50), нақты норма пилоттан кейін шығады (дәл осы себепті дерекқор қосылды — төменде).
- **117 мамандық кілті:** сарапшылық (O*NET логикасына сүйенген), әлі эмпирикалық валидацияланбаған.

**АСА МАҢЫЗДЫ ШЕШІМ (Ерқанатпен келісілген):** платформаны «AI негізіндегі» деп сипаттау керек болса да, **есептеу қабаты (Holland косинус, Климов түзетуі, конвергенция) ешқашан AI-мен ауыстырылмайды** — таза математика күйінде қалады. AI тек ҮСТІНЕ қосылатын «баяндау және сөйлесу» қабаты (система промпттарда, чат интерфейстерде). Бұл — дәлдік пен сенімділіктің негізі, бұзылмауы керек шек.

## Кодтың құрылымы (`kimbolam.html` ішінде)

- `HOLLAND_ITEMS`, `KLIMOV_PAIRS`, `MBTI_ITEMS`, `KLIMOV_TYPES` — сұрақ банктері.
- `SCALE_INTEREST`, `SCALE_AGREE`, `NORM_BASE`, `SECTIONS`, `FLOW` — тест құрылымы.
- `SPECIALTIES` (=`JOBS`) — 117 мамандық `{code, n, subj, hol, w, klim}`.
- `UNIS` — 100 университет (байланыс әлі жоқ).
- Engine: `startTest()`, `renderQ()`, `answer()`/`pickAnswer()`, `next()`/`prevQ()`.
- Scoring: `computeAll()` → `{holland, klimov, mbti}`. `cosine()`, `matchJobs()`.
- Report: `showResults()` — есептің барлық бөлімін DOM id арқылы толтырады, соңында `aiProfile` жаһандық айнымалысын толтырады (AI мүмкіндіктері осыны қолданады).
- `TYPES`, `COMBOS` — Holland типтерінің сипаттамасы.
- AI кеңесші виджеті (`aiFab`, `aiPanel`, `toggleAI()`, `sendAI()` т.б.) — бөлек `<script>` блогында, файлдың басында.

## Тестілеу тәсілі

Браузерсіз функционалды симуляция: document/window stub жаса, script-ті eval ет, `answers` массивін `FLOW` бойынша қолдан толтыр, `computeAll()`/`matchJobs()`/`showResults()` шақыр, `store[id].innerHTML`/`textContent` тексер. Бірнеше профиль расталды (көшбасшы→Менеджмент/HR дұрыс, техник→инженерия дұрыс, қайшы профиль→сенім «төмен» дұрыс).

## Басты шектеулер / есте сақтау керек нәрселер

- **Норма мен кілт әлі пилоттан өткен жоқ** — «дәл тест» емес, «шешім қолдау құралы» деп сипатта.
- **MBTI — ресми емес**, Ерқанаттың авторлық сауалнамасы. UI-де «MBTI» деуге болады (Ерқанат қалайды), ресми құжаттарда «Юнг типологиясы (MBTI дәстүрінде)».
- **Бренд:** «Кім болам?» кириллицада негізгі атау. Latin `kimbolam` техникалық қабатта қолданылады (файл атаулары, кодтағы идентификаторлар).
- Негізгі файл `kimbolam.html` деп аталады (бұрын `kasipzhol.html` болған — Ерқанаттың нақты шешімі бойынша ауыстырылды, себебі жобаның аты «Кім болам?»). **Бұдан былай жобада «kasipzhol» сөзі мүлдем қолданылмайды.**
- **AI-дің екі негізгі этикалық ережесі, барлық жаңа AI мүмкіндікте қайталануы керек:**
  1. Есептеу/сәйкестендіру логикасына AI араласпайды (жоғарыдағы «АСА МАҢЫЗДЫ ШЕШІМ»).
  2. Кез келген жазу-көмекші (эссе, IDP) **coach режимінде** болады — AI студенттің орнына мәтін/шешім жазбайды, тек сұрақ қою/кері байланыс арқылы бағыттайды. Толық нұсқа тек НАҚТЫ сұралғанда және әрдайым «бастапқы нүкте, көшірме» ескертуімен беріледі (қараңыз: `backend/api/essay-coach.js`).

## AI кеңесші (backend қажет, ӘЛІ ДЕПЛОЙ ЕТІЛМЕГЕН)

Нәтиже бетінде 💬 FAB батырма (әрдайым көрінеді, тек тест профилі болса дәлірек сөйлейді). `backend/api/chat.js` — Vercel serverless function, `ANTHROPIC_API_KEY` сервер жағында. Дәлдік/қауіпсіздік: тек нақты тест нәтижесі + 117 мамандық дерекқорына сүйенеді, университет нақты деректің жоқтығын адал айтады, кәмелетке толмағандарға арналған қауіпсіздік ережелері бар.

**Деплой қажет:** `backend/README.md` бойынша Vercel-ге шығару + `kimbolam.html`-дегі `AI_API_URL`-ды нақты деплой URL-іне ауыстыру.

## Мотивациялық эссе көмекшісі (backend қажет, ӘЛІ ДЕПЛОЙ ЕТІЛМЕГЕН, платформаға қосылмаған)

`essay-coach-prototype/index.html` — бөлек прототип (чат панелі + эссе жоба тақтасы). `backend/api/essay-coach.js` — екі режим: **coach** (әдепкі, AI ешқашан толық эссе жазбайды, тек бағыттаушы сұрақ+кері байланыс) және **full_draft** (тек нақты сұралғанда, әрдайым "көшірме" ескертуімен). Кітаптың «NU грантын қалай жеңіп алдым?» тарауымен байланысу үшін жобаланды.

**Деплой қажет:** сол Vercel жобасының `api/` қалтасында (chat.js-пен бірге, қосымша деплой қадамы жоқ) + прототиптегі `AI_API_URL`-ды жаңарту.

## IDP миссия жүйесі (жаңа үлкен идея, backend қажет, ӘЛІ ДЕПЛОЙ ЕТІЛМЕГЕН)

Үлкен концепция: тест — бір реттік сурет («сен кімсің»), миссиялар — уақыт бойы жалғасатын даму жүйесі («сен қайда барасың»), кітаптың III бөлімінің (Тұлғалық даму) сандық жалғасы. **5 деңгей** (кітап бөлімдеріне сай): Тану(тест) → Мақсат қою → Жоспарлау → Әрекет ету → Бағалау. Duolingo-стиль: деңгей-жол, бейдж/XP/серия, робот-серік кейіпкер (дауыстық кіріспедегі роботпен бір дизайн, әлі ортақ компонентке бөлінбеген).

`missions-prototype/mission-1-goal.html` — **бірінші миссия (Мақсат қою) толық айналатын прототип**: кіріспе → AI серігімен диалог (мақсатты нақтылау, AI ешқашан мақсатты өзі қоймайды, тек «қашан?/қалай өлшейсің?/неге маңызды?» деп сұрайды) → студент өз сөзімен мақсат карточкасын толтырады → бейдж/конфетти/XP аяқталу мерекесі. `backend/api/goal-mission.js` — backend (coach-стиль система промпт, essay-coach-пен бірдей ұстаным).

**Әлі істелмегені:** (1) деплой + сынау, (2) мақсат әлі дерекқорға сақталмайды (тек консольге), керек болса `goals` деген жаңа кесте + `save-goal` endpoint қосу керек (`save-result` үлгісі бойынша), (3) қалған 4 деңгей (Жоспарлау/Әрекет ету/Бағалау) жоқ — кітап мазмұны келгенде жобаланады, (4) робот кейіпкер ортақ компонентке бөлінбеген, екі файлда (audio-intro, missions) қайталанып тұр.

## Дауыстық кіріспе (backend қажет емес, локал файл жеткілікті)

`audio-intro-prototype/index.html` — тест алдында робот-серік дауыстап ереже түсіндіреді (72 сұрақ, 15 минут, дұрыс/бұрыс жауап жоқ). Үш сатылы сақтық тізбегі: (1) дайын MP3 файл (`audio/intro-01.mp3`...`intro-12.mp3`, ЕҢ ЖАҚСЫ САПА — Azure `kk-KZ-DauletNeural` дауысы) → (2) браузердің өз дауысы (kk→ru) → (3) субтитр режимі. `generate-audio.js` — бір реттік скрипт, Azure Speech API арқылы MP3 файлдарын жасайды (Ерқанаттың Azure тіркелгісі + кілті керек, `README.md`-де қадамдар жазылған).

**Маңызды факт:** браузердің құрылғыдағы дауысы (`speechSynthesis`) қазақ тілін сирек қолдайды (көп құрылғыда мүлдем жоқ) — сондықтан Azure MP3 жолы негізгі, браузер дауысы тек сақтық.

## Дерекқор (backend қажет, ӘЛІ ДЕПЛОЙ ЕТІЛМЕГЕН, платформаға қосылмаған)

Себебі: адамдар нақты тапсырып жатқандықтан нәтижені сақтау керек болды, әрі бұл — норма/кілтті нақтылайтын пилот дерегінің қайнар көзі. **Supabase** (Postgres), `db-prototype/schema.sql`: екі кесте `classes`, `test_results`.

**Модель — сынып коды арқылы жеңіл байланыстыру** (есептік жазба/пароль жоқ): жеке адам код өрісін бос қалдырады → «жеке»; сыныппен жұмыста алдын ала «сынып» жасалады (`api/create-class.js`) → қысқа код (мыс. `7B-K3M9`) → оқушылар нәтиже бетінде осы кодты да жазады → бәрі автоматты жиналады.

**Backend endpoint-тер:** `create-class.js`, `save-result.js`, `get-class-results.js` (Supabase service_role key арқылы, RLS қосулы, клиенттен тікелей қатынау жабық).

**Сақталатын дерек:** тек нәтиже (Holland/Климов/MBTI + сай мамандықтар) + оқушы аты-жөні (**міндетті**, Ерқанаттың нақты шешімі). **Шикі 72 жауап сақталмайды** (жеңілдету үшін әдейі, дегенмен бұл пилот-валидацияны қиындатуы мүмкін — керек болса қайта қаралуы мүмкін).

**Тексеру:** `db-prototype/test-db.html` — үш endpoint-ті бөлек сынайтын бет. Толық орнату нұсқауы (Supabase жоба ашу, кестелерді SQL арқылы жасау, кілттерді алу — Project Settings → API → Project URL + service_role key) `db-prototype/README.md`-де.

## Барлық жаңа backend-терді деплой ету туралы БІР МАҢЫЗДЫ ЕСКЕРТУ

`chat.js`, `essay-coach.js`, `goal-mission.js`, `create-class.js`, `save-result.js`, `get-class-results.js` — барлығы **бір ғана** `backend/api/` қалтасында, демек **бір ғана Vercel жобасына бірге деплой етіледі** (қосымша жоба ашудың қажеті жоқ). Деплой етілгенде барлық endpoint бір деплой URL-інің астында пайда болады (мыс. `https://kim-bolam-backend.vercel.app/api/chat`, `/api/essay-coach`, т.б.). Әр prototype/негізгі файлдағы `AI_API_URL` (немесе баламасы) сол бір деплой URL-іне сай жаңартылуы керек. Қажетті ENV айнымалылар: `ANTHROPIC_API_KEY` (AI функциялары үшін), `SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY` (дерекқор функциялары үшін).

## Келесі қадамдар (басымдық бойынша)

1. **Барлық backend-ті бір рет Vercel-ге деплой ету** (жоғарыдағы бөлімді қара) — бұл бәрін бірден ашады: AI кеңесші, эссе көмекшісі, IDP миссиясы, дерекқор.
2. **Дерекқорды тірі сынау** (`db-prototype/test-db.html`) — Supabase орнатып, үш endpoint-ті тексеру.
3. **Аудио файлдарды генерациялау** (Azure Speech, `generate-audio.js`) — дауыстық кіріспені толық жұмыс істететін ету.
4. **Кітап мазмұнын алу және миссия жүйесіне интеграциялау** — Ерқанат кітапты жібергенде: 5 деңгейдің қалған 3-еуін (Жоспарлау/Әрекет ету/Бағалау) жобалау, кітаптың нақты жаттығуларын (SWOT, Икигай, «50 әрекет») миссияларға айналдыру.
5. **Университет↔мамандық байланысы** — Ерқанат файл жібергенде интеграциялау.
6. Дерекқор дайын болған соң: `showResults()` соңына `save-result` шақыруын, «Мұғалімге» бетіне сынып коды өрісі мен `get-class-results` интеграциясын қосу.
7. Ұзақ мерзімде: пилот сынақ (50–150 оқушы) — норма мен кілтті нақтылау.
