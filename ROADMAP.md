# Roadmap — MVP-фокус на тесте (текущая итерация)

> Рабочий план на ближайший этап. Долгосрочный бэклог (интеграция книги, университеты и т.д.)
> — в [CLAUDE.md](CLAUDE.md), раздел «Келесі қадамдар». Этот файл — что решили и сделали
> именно в текущей сессии, чтобы не потерять контекст.

## Контекст / решение

Фронт и бэк разрабатывались параллельно по многим направлениям (AI-советник, эссе-коуч,
IDP-миссии, каталог мамандықтар, университеты, кабинет учителя/родителя), но ни одно
ещё не в проде. Решили: **не распыляться**, свести MVP к одному — «зашёл → прошёл тест →
увидел результат → результат сохранился». Всё остальное убрано с глаз пользователя
(код не удалён, просто не подключён к навигации), чтобы включать по мере готовности.

## Статус этапов

### ✅ Локальный backend (SQLite + Docker) — сделано
- `backend/db/` — абстракция: `sqlite.js` (локально) / `supabase.js` (прод), выбор через
  `DB_DRIVER` в `.env`. При деплое меняется только env, код `api/*.js` не трогаем.
- Закрыта дыра безопасности: раньше один `classCode` (который получают ученики) открывал
  и сохранение результата, и просмотр результатов всего класса — то есть один ученик мог
  вытащить данные одноклассников. Теперь `create-class` выдаёт `code` (ученику) и отдельный
  секретный `teacherCode` (только учителю, только он открывает `get-class-results`).
- CORS теперь через `ALLOWED_ORIGIN` (не `*`), rate-limit на все 6 эндпоинтов
  (`backend/lib/http.js`) — проверено вживую (429 после лимита).
- `docker-compose.yml`: `backend` (Node+Express+SQLite, :3000) + `frontend` (nginx статика, :8080).
- Docker Desktop на этой машине — только версия **4.43.0** (macOS 13.7.8 слишком старая для
  актуальных сборок, которые требуют macOS 14+). Если переустанавливать — качать именно эту
  версию, а не «latest».

### ✅ MVP-урезание фронта — сделано
- Шапка (`kimbolam.html`): убраны все ссылки навигации (Мамандықтар/Университеттер/
  Мұғалімге/Ата-анаға/Блог/Кіру/Тіркелу) — остались только лого + кнопка «Тестті бастау».
  Заодно ушли и уродливые `alert()`-заглушки.
- Страницы catalog/unis/teacher/parent **остались в коде**, просто недостижимы из UI.
- AI FAB (💬) скрыт по умолчанию (`class="hidden"`) — backend для него ещё не задеплоен,
  показывать нерабочую кнопку не стали. Включить обратно — убрать один класс.

### ✅ Сохранение результата теста — сделано и проверено
- В отчёте — поле «Сынып коды (болса)» рядом с именем.
- `showResults()` собирает `resultPayload` (Holland/Климов/MBTI/топ-5 мамандық) и по `blur`
  с поля имени один раз за сессию вызывает `save-result` (`trySaveResult()` в `kimbolam.html`).
- Проверено end-to-end через браузер: прошёл тест → ввёл имя+код класса → запись появилась
  в SQLite ровно один раз (повторный blur не дублирует) → `get-class-results` с teacherCode
  её возвращает, с code ученика — нет (защита работает).

### ⏳ Готово к деплою, ждём команду — НЕ выполнено
Ничего не задеплоено намеренно. Когда будет команда:
1. Создать проект в Supabase, прогнать `db-prototype/schema.sql` (уже с `teacher_code`).
2. В Vercel (`backend/`) выставить env: `ANTHROPIC_API_KEY`, `DB_DRIVER=supabase`,
   `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `ALLOWED_ORIGIN` (реальный домен).
3. Заменить `PROD_API_BASE` (плейсхолдер `https://YOUR-DEPLOY-URL.vercel.app`) на реальный
   URL в 4 файлах: `kimbolam.html`, `essay-coach-prototype/index.html`,
   `missions-prototype/mission-1-goal.html`, `db-prototype/test-db.html`.

### ⏳ Git — репозиторий инициализирован, коммита нет
`git init` сделан, `.gitignore` настроен (не пускает `.env`, `*.db`). Первый коммит —
только по прямому запросу.

## Осознанно отложено (код есть, не подключено)

AI-советник, эссе-коуч, IDP-миссии, реальные данные каталога/университетов (нет данных

## Срочный запуск (перекрывает пункт про Supabase/Vercel выше)

Реальные ученики начинают сдавать тест раньше, чем готов Vercel/Supabase-деплой. Решение:
**Google Sheets вместо базы данных**, без какого-либо деплоя бэкенда.

- Перед тестом теперь обязательный экран (`#gate` в `kimbolam.html`): класс, ФИО, промокод.
  Все кнопки «Тестті бастау» ведут туда (`goGate()`), а не сразу в тест.
- Промокод — **одноразовый**, проверяется через Google Apps Script Web App (`checkCode`),
  расходуется при отправке результата (`submitResult`). Логика и инструкция —
  `google-sheets-integration/` (`Code.gs`, `README.md`, `promo-codes.csv` — 500 готовых кодов).
- Результат теперь пишется в Google Sheets (`submitToSheet()`), а не в SQLite/Supabase —
  старый backend (`backend/`) не удалён, просто не используется, пока не решим мигрировать обратно.
- Отчёт скачивается как PDF в один клик (`downloadPDF()`, html2pdf.js) — раньше был только
  `window.print()`.
- **Проверено локально** end-to-end через мок-сервер (checkCode/submitResult/PDF-экспорт,
  8 страниц) — код рабочий. Также **проверено вживую на реальном Apps Script** пользователем
  (после серии проблем с доступом деплоя — см. ниже) — `checkCode`/`submitResult` работают.

### ✅ Раздел «Кеңірек қарасаң» (AI-подсказка сверх 117 специальностей)

Пользователь попробовал сравнить наш расчётный топ-5 с ответом Gemini/Claude в чате и
предложил встроить более широкий, не ограниченный 117 официальными специальностями,
AI-список прямо в отчёт (не через AI-советника — тот отложен). Договорились:
**расчётный топ-5 остаётся как есть** (это официальный результат), рядом добавлен отдельный,
явно подписанный блок **«Кеңірек қарасаң»** — AI-мнение, не расчёт, вызывается только по
кнопке (не автоматически, чтобы не тратить лимит API на каждого тестирующегося).

- `kimbolam.html`: новая секция в отчёте после раздела 07 (`loadBroadSuggestions()`,
  кнопка `#broadBtn`, вывод в `#broadList`).
- `google-sheets-integration/Code.gs`: новый `action=suggestBroad`, дергает Gemini API
  через `UrlFetchApp` (ключ `GEMINI_API_KEY`, хранится только на сервере).
- **Блокер**: у пользователя пока нет рабочего Gemini/Anthropic API-ключа — AI Studio
  выдаёт «suspicious request» при создании ключа даже после ручного создания Google Cloud
  проекта. Код готов и проверен локально (мок вернул тестовый список, отрендерился корректно),
  но с реальным Gemini не тестировался — как только ключ появится, вписать в `Code.gs`
  (`GEMINI_API_KEY`) и передеплоить (New version).

### ✅ Найден и частично исправлен баг в данных 117 специальностей

Пользователь прошёл тест сам и получил в топ-8 явно нерелевантные специальности (Мейрамхана
ісі, Кітапхана ісі, Туризм, Халықаралық экономикалық қатынастар, Құқық) наравне с релевантными
(Менеджмент, Еңбек дағдылары). Диагностика показала: **это не баг в коде подбора** (косинус +
Климов считают верно), а **дублирующиеся Holland-векторы** у разных по сути специальностей —
например `B044 Менеджмент` и `B049 Құқық` имели буквально идентичный вектор `{E:3,C:2,S:1}`,
`B048 Еңбек дағдылары` и `B093 Мейрамхана ісі` — идентичный `{E:3,S:2,C:1}`. При таком
совпадении косинусное сходство физически не может их различить.

Исправлено точечно (в `SPECIALTIES` в `kimbolam.html`, ~строки 1269-1339): `B049 Құқық`
(ECS→EIS), `B048 Еңбек дағдылары` (ESC→SEC), `B093 Мейрамхана ісі` (ESC→ESR),
`B141 Халықаралық экономикалық қатынастар` (ECI→EIC), `B046 Қаржы` (CEI→CIE) — каждое
изменение обосновано реальной психологией профессии (Право = Investigative-анализ, а не
Conventional-делопроизводство; HR = люди прежде всего; Ресторанный бизнес = Realistic-ручные
операции, а не офисная системность). Проверено синтетическим E/C-профилем в браузере:
Менеджмент/HR теперь чётко ранжируются выше Құқық/Мейрамхана ісі, а Мейрамхана ісі и Туризм
выпали из топ-8.

**Известное ограничение, НЕ исправлено**: то же самое дублирование векторов массово
встречается в остальной части списка (например ~30 технических специальностей имеют
идентичный `{R:3,I:2,C:1}`) и в паре мест (`B008`, `B043`) осталась мягкая, менее очевидная
переоценка через бонус Климова — это требует отдельного, более полного прохода по всем 117
записям с участием Ерқанат (как эксперта), а не одностороннего решения.
