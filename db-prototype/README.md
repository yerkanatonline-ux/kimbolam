# Дерекқор — орнату нұсқаулығы

## Не үшін керек?
Оқушылардың тест нәтижесін сақтау (әсіресе — бұл жиналған дерек кейін норма мен
мамандық кілтін нақтылайтын пилот дерегі болады). Сонымен қатар мектеп/сыныппен
жұмыс істегенде нәтижелерді бір жерден көру мүмкіндігі.

## Модель: сынып коды
- **Жеке адам** (сырттан): ешбір кедергісіз тестке кіреді, код өрісін бос қалдырады.
- **Сыныппен жұмыс**: сіз/мұғалім алдын ала «сынып» жасайды → қысқа код шығады
  (мыс. `7B-K3M9`) → оқушыларға таратасыз → олар нәтиже бетінде осы кодты да
  жазады → барлығы автоматты түрде бір топқа жиналады.
- Есептік жазба/пароль жүйесі жоқ (MVP үшін жеткілікті, кейін қажет болса қосамыз).

## Орнату қадамдары

### 1. Supabase жобасын жасау
1. https://supabase.com — тіркеліңіз (GitHub арқылы кіруге болады, тегін деңгей жеткілікті)
2. "New project" — атын қойыңыз (мыс. `kim-bolam`), регион таңдаңыз, дерекқор паролін ойлап табыңыз
3. Жоба дайын болғанша 1-2 минут күтіңіз

### 2. Кестелерді жасау
1. Supabase дашбордында → SQL Editor
2. `db-prototype/schema.sql` файлының мазмұнын көшіріп қойыңыз → Run

### 3. Кілттерді алу
1. Supabase дашбордында → Project Settings → API
2. Мыналарды көшіріп алыңыз:
   - **Project URL** (мыс. `https://xxxxx.supabase.co`)
   - **service_role key** (⚠️ бұл құпия кілт, тек backend-те қолданылады, ешқашан клиентте/GitHub-та жарияланбайды)

### 4. Vercel-ге қосу
`backend/` қалтасында (AI кеңесші тұрған жерде):
```
vercel env add SUPABASE_URL
vercel env add SUPABASE_SERVICE_ROLE_KEY
```
Немесе Vercel дашбордында → жобаңыз → Settings → Environment Variables арқылы да қосуға болады.

Дерекқор пакетін орнату:
```
cd backend
npm install
vercel --prod
```

### 5. Тексеру
`db-prototype/test-db.html` файлын ашыңыз, үш URL-ды (`CREATE_CLASS_URL`,
`SAVE_RESULT_URL`, `GET_CLASS_URL`) нақты деплой сілтемеңізге ауыстырыңыз, содан кейін:
1. «Сынып жасау» — код шығуы керек
2. «Үлгі нәтижені сақтау» — сол кодпен бір оқушы нәтижесін сақтаңыз
3. «Нәтижелерді көрсету» — сол кодпен іздеп, кестеде көрінуі керек

Үшеуі де жұмыс істесе — дерекқор дайын, содан кейін ғана негізгі платформаға
(`showResults()` соңына `save-result` шақыруын, «Мұғалімге» бетіне `get-class-results`
интеграциясын) қосамыз.

## Әлі қосылмаған (келесі қадам)
- `kimbolam.html`-дың өзі әлі бұл endpoint-терді шақырмайды — сіз сұрағандай, алдымен
  осы бөлек бетте тексереміз.
- «Мұғалімге» беті әлі нақты дерекқордан оқымайды (қазір статикалық үлгі дерек көрсетеді).
