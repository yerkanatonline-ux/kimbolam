# Google Sheets арқылы жедел іске қосу (Vercel/Supabase-сіз)

Бұл — толық backend (Vercel + Supabase) дайын болғанша қолданылатын жылдам жол. Тек Google аккаунт керек, ешбір деплой/консоль қажет емес.

## 1-қадам: Google Sheet жасау

1. [sheets.google.com](https://sheets.google.com) → жаңа кесте ашыңыз, атын мыс. «Кім болам? — промокод және нәтижелер» деп қойыңыз.
2. Кестенің төменгі жағында 2 бет (tab) жасаңыз: **Codes** және **Results** (әдепкі "Sheet1"-ді "Codes" деп өзгертіп, жаңа бет қосып "Results" деп атаңыз).
3. **Codes** бетінің 1-жолына тақырып жазыңыз: `code | used | used_by | used_at` (A1, B1, C1, D1 ұяшықтарына).
4. `promo-codes.csv` файлын ашып, А бағанындағы (`code`) 500 промокодты көшіріп, Codes бетінің A2 ұяшығынан бастап қойыңыз (Google Sheets CSV-ды авто-бөледі, немесе File → Import → осы csv файлды Codes бетіне импорттаңыз).
5. **Results** бетінің 1-жолына тақырып жазыңыз: `timestamp | class | name | code | hollandCode | klimovTop | mbtiType | confidence | topSpecialties`.

## 2-қадам: Apps Script орнату

1. Кесте ішінде: **Extensions → Apps Script**.
2. Ашылған терезедегі дайын кодты өшіріп, осы қалтадағы `Code.gs` файлының мазмұнын толық қойыңыз.
3. Жоғарыдағы **Save** (дискета белгісі) басыңыз.
4. **Deploy → New deployment**.
5. Тісті дөңгелек ⚙️ басып, түрін **Web app** етіп таңдаңыз.
6. **Execute as**: `Me` (өз аккаунтыңыз). **Who has access**: `Anyone`.
7. **Deploy** басыңыз → Google рұқсат сұрайды (өз аккаунтыңызбен рұқсат беріңіз — "Advanced" → "Go to (жоба аты)" керек болуы мүмкін, бұл қалыпты, себебі скрипт сіздің кестеңізге жазады).
8. Шыққан **Web app URL**-ды көшіріп алыңыз (`https://script.google.com/macros/s/.../exec` түрінде).

## 3-қадам: kimbolam.html-ге қосу

`kimbolam.html` файлында (шамамен 790-жол маңы) мына жолды тауып:

```js
const SHEETS_WEBAPP_URL = 'https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec';
```

`YOUR_SCRIPT_ID` бөлігін 2-қадамда алған нақты URL-ге ауыстырыңыз.

## Тексеру

`kimbolam.html`-ден толық тест өтіп көріңіз (промокодтардың бірін еніп): Codes бетінде сол код `used=TRUE` болып белгіленуі, ал Results бетінде жаңа жол пайда болуы керек.

## (Міндетті емес) 4-қадам: «Кеңірек қарасаң» бөлімі үшін Gemini кілті

Есептің «07» бөлімінен кейін «Кеңірек қарасаң» деген қосымша блок бар — ол 117 ресми
мамандықпен шектелмей, AI-дің жалпы ұсынысын көрсетеді (тек батырма басылғанда, автоматты
емес). Бұл — есептелген нәтиженің бөлігі емес, қосымша идея ретінде белгіленген.

1. [aistudio.google.com/apikey](https://aistudio.google.com/apikey) → **Create API key**
   (Google Cloud жобасы керек болса, алдымен [console.cloud.google.com/projectcreate](https://console.cloud.google.com/projectcreate)
   арқылы бос жоба жасап алыңыз, содан кейін соны таңдаңыз).
2. Шыққан кілтті (`AIza...` басталады) `Code.gs`-тегі мына жолға қойыңыз:
   ```js
   const GEMINI_API_KEY = 'YOUR_GEMINI_API_KEY';
   ```
3. **Deploy → Manage deployments → ✎ → Version: New version → Deploy** (кілт қолданысқа енуі үшін).
4. Кілт қойылмаса — батырма басқанда «GEMINI_API_KEY орнатылмаған» деген қате көрсетеді,
   бірақ бұл негізгі тестке (промокод/сақтау/PDF) кедергі жасамайды.

## Ескерту

- Промокод **бір рет қана** жұмыс істейді — қолданған соң қайта енгізуге болмайды.
- Бір мезгілде бірнеше оқушы жіберсе де қауіпсіз — `LockService` қатарды сериялайды (сирек жағдайда 10 секундтан көп кідіріс болуы мүмкін, бірақ деректер жоғалмайды).
- Бұл — уақытша шешім. Толық backend (Supabase) дайын болғанда, `docs`/`ROADMAP.md`-де жазылған жоспар бойынша ауыстырылады.
